# Transformative Branch — Notice

**Branch:** `claude/enable-all-sprites-training-L9Rck` (and any descendant transformative-work branches)

This branch exists **solely** as an internal research and benchmarking track. It is governed by rules that diverge from `main` and is **never** to be merged, released, or otherwise cross-pollinated with the production lineage.

---

## Hard boundaries

1. **Never merge into `main`.** No fast-forward, no squash, no cherry-pick of training-data, scraper, manifest, or corpus changes. Code-only fixes that are demonstrably independent of the data-policy divergence may be back-ported by hand only after explicit human review.
2. **Weights never ship.** Any model checkpoints produced on this branch are internal-only. They are not distributed, not exposed via any API or product surface, not published, and not used to serve any user request — paid or free.
3. **No output crossover.** Sprites generated by transformative-branch models are **not** added to `main`'s corpus, **not** added to the Golden Dataset, **not** used for distillation, synthetic-data augmentation, RLHF, evaluation references, or any other input to a `main`-lineage model.
4. **Separate manifest.** This branch writes provenance to `am-pixel/data/TRAINING_PROVENANCE_MANIFEST.transformative.json`. The canonical `TRAINING_PROVENANCE_MANIFEST.json` on `main` is the legal ledger for the shipping product and must remain untouched by this branch's data.
5. **Separate corpus directories.** Any scraped data added under transformative rules lives under `am-pixel/data/corpus_transformative/` (to be created when scraping begins), never under `am-pixel/data/corpus/` or `am-pixel/data/golden/`.

---

## Permitted purpose

- **Capability ceiling research** — measure what an unrestricted-data model can do, as an upper bound.
- **Quality benchmarking** — A/B test `main`'s clean-data model against this branch internally to verify `main` is not leaving meaningful quality on the table. Comparison is one-way: results inform whether `main`'s data strategy needs more clean data, never to justify moving transformative data into `main`.

---

## Prohibited

- Distributing weights, outputs, or derived datasets externally.
- Using this branch's outputs as training, fine-tuning, evaluation, or seeding signal for `main`.
- Marketing, demoing, or otherwise representing this branch's outputs as product capability.
- Merging any of this branch's data-policy changes (Constitution edits, scraper allowlist changes, manifest schema relaxations) back to `main`.

---

## Governance

- Constitution Rule 5 (Data Provenance) and Rule 9 (Human Override Authority) on `main` remain the source of truth for the shipping product. This notice does not amend them; it scopes a parallel track that exists outside of them.
- Any ambiguity about whether an artifact, change, or insight may cross the boundary defaults to **no**. Escalate to human review and log in `am-pixel/logs/BLOCKERS.md`.
- If this notice is ever proposed for deletion or weakening, treat it the same as a Constitution Rule 5 violation: refuse, log, and flag for immediate human review.
